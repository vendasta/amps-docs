### Step-by-step

| No. | Stage |
| --- | --- |
| 1 | [Partner Center](https://partners.vendasta.com/dashboard) → PID (top-left) → type **COP2** → select first result |
| 2 | On COP2 home: if count is **0**, nothing to do; if **≥1**, click the number → accounts list |
| 3 | Find accounts created in the **last 4 days** (creation date on the right) |
| 4 | Open each new account → if **Google Business Profile Management** already active, skip; else continue |
| 5 | **View All Data** (top right) → scroll to **business category** |
| 6 | Set category from business name (see table below) |
| 7 | **Save** → **Back** to account screen |
| 8 | **Duplicate check:** copy address → PC **Manage Account** search → same address + **similar** name = possible duplicate (e.g. Co-op Agro + Co-op Cardlock ✓; Co-op Agro + Co-op Home ✗). If duplicate suspected: **stop**, do not activate, note GBP Management task, reassign to comms. Else continue |
| 9 | **Products** → **Activate Products** _(screenshots TBD when new co-op claim)_ |
| 10 | **Packages** → **Retail Package** |
| 11 | Confirm products: Reputation Management, Task Manager, Local SEO, Listing Sync Pro (Monthly), Google Business Profile Management |
| 12 | Refresh if any missing |
| 13 | If still missing, return to **step 4** and adjust **Business Category** |
| 14 | **Retail Package** → **Proceed to Next Step** |
| 15 | **N/A** for all contact/sales person info |
| 16 | **Next Step** |
| 17 | **Agree** to terms → **activate** |
| 18 | Refresh until all products **active** (not pending) |

### Claim & Verify GBP / Listing Sync / Transfer Access / Confirm Posts / Send Confirmation

Once the listing is claimed and verified, add summarizing task notes and close the claim task.

**Turn on Listing Sync:** Select the task, open "Local SEO" (green icon) from the task pane, then "Listing Sync". Select "Sync to Google" (or "Sync to Fix Google"); when the sidebar opens, select "Sync to Google" again. If we have ownership: add *Syncing as a user on the listing in GBP dashboard, then in syncing page choose your personal Vendasta account → allow → choose *Syncing location group → Select the listing. If we only have management access (listings@yourdigitalagents.com): choose "Marketing M" or "use another account" and log in with listings@yda; choose "Marketing M" location group, find the listing (Ctrl+F), Select → activate syncing. If you added *Syncing, remove it from the listing in GBP dashboard after syncing.

**Transfer Access to Business Gmail:** Invite the email from the fulfillment form as owner unless otherwise instructed. Leave a public task note with the email and mark complete.

**Confirm Number of Monthly Posts:** Account page → Products & Services. GBP Optimization includes 1 post/month; add-ons "Additional Google Business Profile Post: 1x Month" add more (check blue icon number 2–7 or one add-on = 2 total). Update task notes with post count; mark "in progress" until listing is claimed and verified.

**Set up recurring project:** Vendor Center (VMKA) → Work Orders → find original order → kebab menu → Create project → search "Google Business Profile Optimization:" → Create Project. Add public note "Project created" and mark complete.

**Send Confirmation Email:** Reassign task to Comms assignee. Public notes: summary and that we're ready to send final confirmation. Private notes: macro or "send on existing thread [ZD ticket]". Set due date to next day, mark "In progress"; Comms marks complete when sent.

## Google Support Follow-Up Process

| No. | Stage |
| --- | --- |
| 1 | Email ticket via [listings@yourdigitalagents.com](mailto:listings@yourdigitalagents.com). |
| 2 | Log Support ticket # in VTM; email client/partner that Google was contacted. |
| 3 | Google usually replies in **~3 business days**; reply promptly; CE sends partner follow-up when you follow up with Google. |
| 4 | **No response in 2 weeks:** escalate (often new ticket referencing old #). |
| 5 | If Google succeeds while alternate paths run, remove duplicate listing if appropriate and verification still pending. |

**Note:** Use public task notes for information that clients can see. Sensitive information such as references to "partner", "Vendasta", "branded email", etc., or information that the client shouldn't know can go in the private notes. Keep in mind that the partner can always see the private notes.
