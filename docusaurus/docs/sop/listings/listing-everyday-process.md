## COP2

**Short version:** Go into COP2 in Partner Center → Look for new accounts → Add business category → Activate retail package.

**Step-by-step:**

1. Go to [Partner Center](https://partners.vendasta.com/dashboard). Click on the PID name in the top left corner, then type "COP2" into the box that pops up. Select the first option from the list.

![](/img/sop-listings/listing-everyday-16.png)
2. Look at the number on the front page. If the number is zero, there is nothing to do. If the number is 1 or more, click on the number to go to the accounts screen.

![](/img/sop-listings/listing-everyday-2.png)
3. Look for any accounts that have been created within the last four days. You can see the creation date on the right-hand side of the table.

![](/img/sop-listings/listing-everyday-24.png)
4. Click on any account that's been created within the last four days. Check to see if Google Business Profile Management is active on the account. If it is active, there is nothing to do. Move on to the next account.

![](/img/sop-listings/listing-everyday-18.png)
5. If it's not active, select "View All Data" at the top right of the screen, then scroll down to add the business category.

![](/img/sop-listings/listing-everyday-23.png)
![](/img/sop-listings/listing-everyday-20.png)
6. You can find the business category in the business name. Reference:
   * Co-op Gas Bar = Gas Station
   * Co-op Cardlock = Alternative Fuel Station
   * Co-op Liquor = Liquor Store
   * Co-op Pharmacy = Pharmacy
   * Co-op Agro = Agricultural cooperative
   * Co-op Food = Grocery
   * Co-op Home = Hardware Stores
   * Co-op Bulk Petroleum = Fuel Supplier
7. Scroll to the bottom and select "Save", then click the Back arrow at the top to return to the account screen.

![](/img/sop-listings/listing-everyday-21.png)

8. Check to see if there is already an existing account active.

* a. Copy the address of the new account.
* b. Under "Manage Account" on PC, search for the address.
* c. Check to see if there is already an account with the same address and with a similar name.
  * i. There can be multiple listings at the same address (EX. Co-op Agro & Co-op Card Lock ✓ BUT something like Co-op Agro and Co-op Home and Agro is similar ✗)
* d. If similar accounts are found that could be duplicates, halt work on the project, do not activate any products, add a task note to the GBP Management project that the account is possibly a duplicate, and re-assign the task to the assigned comms person. She will check in with co-op corporate to confirm. If no duplicates are found, move on to step 9.

9. In the Products tab, select "Activate Products" (SCREENSHOTS WILL COME FOR THIS PORTION ONCE THERE'S A NEW CO-OP CLAIM TO ACTION).

10. Click into the Packages tab and look for the Retail Package.
11. There should be the following products: Reputation Management, Task Manager, Local SEO, Listing Sync Pro (Monthly), Google Business Profile Management.
12. If you do not see all of these products, attempt to refresh the page.
13. If you still do not see them all, go back to Step 4 and adjust the Business Category.
14. Click the Retail Package and select "Proceed to Next Step".
15. Type N/A for all contact/sales person info.
16. Select Next Step.
17. Select Agree to the terms and activate.
18. Refresh the page until all the products are active and are no longer pending.

### Non-high-touch claims

Process is separated into GBPV C, GBPO, LC, LCMP.

#### Google Business Profile Verification Claim

1. Assign the project and all tasks to yourself.
2. Check for fulfillment form in Vendor Center (Work Orders table).
3. If the fulfillment form hasn't yet been filled out:
   * Leave a note on the first task ("Check for Existing Listing and Access") saying the fulfillment form hasn't been filled out.
   * Assign task to relevant Comms assignee.
   * Change due date to the next business day.
   * Bump dates on all other tasks project by four business days (leave these assigned to you).
4. If the fulfillment form has been filled out, continue to step 5.
5. Using the information on the fulfillment form, complete the tasks.
6. **Check for Existing Listing Access:** If there is an existing listing noted on the fulfillment form, note the link in these task notes. If there is no existing listing, mention this in the task notes. Mark this task as completed.
7. **Claim Verify GBP:** Use this task to track the work being completed for the Google claim. If we had to request access, that info should be noted here. Or if we have access but are waiting for the client to complete the verification, that info should go here and the task should be assigned to the Comms assignee. Any updates on Google tickets or reinstatements should be noted in the task notes. If we already have access to the listing and it is verified, make note of that and mark the task as completed.
8. This task will likely be the longest ongoing task of the project as it is the "communications base" between you and the Comms team. Keep in mind that most notes should go in **Public task notes** so that clients can see the updates. Sensitive information (e.g. "partner", "Vendasta", "branded email", or information that the client shouldn't know) can go in the **private notes**. Keep in mind that the partner can always see the private notes.
9. Once the listing is claimed and verified, add summarizing task notes and close this task.
10. **Turn on Listing Sync:** Listings can be synced even if they're not yet verified. Select the task then open "Local SEO" (green icon) from the task pane. Once Local SEO opens, select "Listing Sync" from the sidebar. On this page, select "Sync to Google" (or possibly "Sync to Fix Google"). When the right-hand sidebar opens, select "Sync to Google" again.
    * **If we have ownership access to the listing:** Open the listing in the GBP dashboard and add *Syncing as a user on the listing (owner or manager). Go back into the syncing page, which should now be asking you to "Choose an account". Select your personal Vendasta account, which should be the first option on the screen. When prompted, select "allow". Choose *Syncing from the list of location groups that will now show up. Locate the listing you want to sync, and click "Select" on the right-hand side.
    * **If we only have management access to the listing (through listings@yourdigitalagents.com):** Go back into the syncing page, which should now be asking you to "Choose an account". If one of the options on this page is "Marketing M" (listings@yourdigitalagents.com), select this then click "allow" on the next page. If the only option given is your personal Vendasta account, select "use another account". Log in with the credentials for listings@yda when prompted. Click "next" when prompted, then select "allow". Choose "Marketing M" from the list of location groups that will now show up (will be the very first one). Locate the listing you want to sync, and click "Select" on the right-hand side. The list of businesses will be very long—use Ctrl+F to find the business name. You will likely need to select "load more" from the bottom of the list several times. You'll need to continually use Ctrl+F as you load more. Once the listing has been selected, you'll be redirected to the Listing Sync main page and a pop-up will appear asking you to "activate syncing". Select this option. Listing is now synced. If we added the *Syncing location group in step 6.a, go back into the GBP dashboard and remove this location group from the listing.
11. **Transfer Access to Business Gmail:** The fulfillment form will instruct you as to which email address to invite to join the listing. Invite this user as an owner on the listing unless otherwise instructed. Leave a public task note stating the email address you sent access to, then mark this task as complete. If there is no one to give access to, make a note of this and mark as complete.
12. **Send Confirmation Email:** Once all tasks are complete, reassign this task from yourself to the Comms assignee. In the public notes, give a summary of the work that's been completed and state that we are ready to send the final confirmation. In the private notes, either fill out the macro with the relevant info or say that we can send the confirmation on the existing thread [XXXXXX]. You can replace the Xs with the number of the Zendesk ticket if it was mentioned under another task. If no ZD ticket number was mentioned, just fill out the macro. Change the due date to the following day and mark the task as "In progress". The Comms assignee will mark it as complete once they've sent the confirmation.

#### Google Business Profile Optimization

1. Assign the project and all tasks to yourself.
2. Check for fulfillment form in Vendor Center (Work Orders table).
3. If the fulfillment form hasn't yet been filled out:
   * Leave a note on the first task ("Check for Existing Listing and Access") saying the fulfillment form hasn't been filled out.
   * Assign task to relevant Comms assignee.
   * Change due date to the next business day.
   * Bump dates on all other tasks project by four business days (leave these assigned to you).
4. If the fulfillment form has been filled out, continue to step 5.
5. Using the information on the fulfillment form, complete the tasks.
6. **Check for Existing Listing Access:** If there is an existing listing noted on the fulfillment form, note the link in these task notes. If there is no existing listing, mention this in the task notes. Mark this task as completed.

Or if we have access but are waiting for the client to complete the verification, that info should go here and the task should be assigned to the Comms assignee. If this add-on is present, double check how many there are. If there are multiple, there will be a blue icon with a number from 2 to 7 at the top-right of the product icon. If the add-on is present but there's no number on it, it is just one add-on—so the client will receive 2 posts total. Once you know the number of posts, update the task notes with the number of posts. Mark the task as "in progress". Do not continue on to step 13 until the listing is claimed and verified.

Once you've found the correct order (double-check AGID and Product), click the three-dot kebab menu and select "Create project". Once the project is created, add a new public task note saying "Project created", then mark the task as complete.

In the public notes, give a summary of the work completed and state that we are ready to send the final confirmation. ## Google Support Follow-Up Process

1. When reaching out to Google Support is required for a listing that is not cooperating, submit an email ticket through [listings@yourdigitalagents.com](mailto:listings@yourdigitalagents.com).
2. Note down the Support ticket number in VTM task notes and notify the client/partner over email that you have reached out to Google Support.
3. Google typically responds in three business days. Keep an eye out for their response so that you can reply. When you follow up with Google, also let the CE team send a follow-up email to the account's client/partner so they are aware we are continually working on the account.
4. If Google has not responded to your ticket in a maximum of two weeks, we will seek an alternate option to proceed with the project based on the listing scenario (most likely submitting another ticket and referencing the old ticket number in the new ticket submission).
5. In any way, if we receive a successful response from Google while we explore the alternate options, we can remove the listing we created (if it was the best option) if verification is pending.

**Note:** Use public task notes for information that clients can see. Sensitive information such as references to "partner", "Vendasta", "branded email", etc., or information that the client shouldn't know can go in the private notes. Keep in mind that the partner can always see the private notes.
