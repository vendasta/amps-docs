## QA: Onboard Call {#qa-onboard-call}

_**If there is adequate information in the fulfillment form, determined in step 1, the onboard call will be skipped and we will start straight with the build.**_

- The member joining the onboard call will be assigned the task _of Reviewing Assets_ to prepare for the call before it is booked. Review the notes from the Team lead in the _Fulfillment Audit_ task and note down any additional assets you need that will be discussed in the onboarding call.
    2. Close the task with notes
    3. Follow the _Onboard Checklist_ during the call. 
       1. When joining the onboard/consultation call, ensure that the call is recorded with a transcript. Enable Gemini to take notes for automatic summarization. Share the summary documentation link with the client engagement and the fulfillment team (website designer and content writers). Instructions on how to enable Gemini notes can be found [here](https://support.google.com/meet/answer/14754931?sjid=18296887865580154859-NC).

  - **Onboarding Checklist** \- ensure all these points are mentioned in the onboarding call
         1. **🕒 Turnaround Times:**
            1. Confirm if they want the first 3 pages delivered in the first mock or up to 10 pages.

3-page mockup delivered in 5–7 business days after onboarding call.

Additional pages (after mockup approval) will be delivered in 5–10 business days, depending on the number of pages.

    - ✏️**Revisions:**
           1. Two major rounds of revisions are included at no additional cost.
           2. Additional revisions beyond the two rounds may impact timelines and scope.
        2. **🔒 Backend Access:**
           1. Access to the backend is restricted using a User Role Plugin until the site is finalized and launched.
        3. **📅 Due Dates & Edit Management:**
           1. Once a due date is confirmed, no new pins should be submitted in Markup.
           2. Any new edits submitted via email after that point may result in a new due date, depending on the complexity of the request.
    1. Summarize all notes in the onboarding call and summary task, and confirm/communicate the due date.

  1. QA: Communicate the First Mock Due Date
     1. Confirm the due date of the First Mock based on this guide:
     2. Page Count
        1. 3-page build: 5 business days
        2. 4-7 pages: 7 business days
        3. 8-10 pages: 10 business days
        4. Additional pages above 10:
           1. Up to 3: add 3 business days
           2. 4-10 pages: add 5 business days
           3. **If there are 20+ pages, build the pages in 10-page phases
        5. **If there’s not adequate content for the pages, note that it will be built with dummy text until it is provided.**
        6. **For the first mock, we’ll deliver the first 3-pages or up to 10 pages for the first mock, depending on the client's requests.**
        7. **Up to 2 business days can be added to the timeline for Divi Plus themes. Ensure you are communicating the reason for the additional days when communicating the due date to CSS.**
     3. Hourly Charge
        1. 1-5 hours: add 2 business days/hourly charge
        2. More than 6 hours: The developer will determine the due date based on the complexity of the request
     4. IDX integration: 
        1. Add 6 business days
     5. Advanced eCommerce: 
        1. Add 5 business days
     6. **Communicate the due date with communications, designer, @ContentTeam, content writer, in the project comments. The content team will build out the content with the information provided by the next business day.**
     7. **When completing the Send Confirmed Due Date task, please add a note communicating the due date and then assign the task to the designated Client Solution Specialist contact so they can send the due date to the client. Please do not close out the task at this stage.**

## Designer: Divi Theme Options {#designer-divi-theme-options}

- **General**
       1. Upload Logo. no larger than **200px wide** , 72 ppi
          1. Now would be a good time to upload the Favicon as well.
       2. Fixed Navigation Bar - Enabled
       3. Color Picker Default Palette - This will change the default colors in any of the modules for color changes to the template.  
![](/img/sop-website-services/templated-website-build-divi/19.png)

**To run multiple CRM forms on the same page use below script!!**

```html
<script>

(function() {

const placeholders = document.querySelectorAll('script[id="__custom_form_widget"]');

if (placeholders.length <= 1) {

return;

}

const firstContainer = placeholders[0].parentElement;

if (!firstContainer) {

return;

}

let hasBeenFixed = false;

const fixFormLocations = () => {

if (hasBeenFixed) return;

const generatedForms = document.querySelectorAll('form.form-widget');

if (placeholders.length === generatedForms.length) {

for (let i = 1; i < placeholders.length; i++) {

const correctPlaceholder = placeholders[i];

const formToMove = generatedForms[i];

if (correctPlaceholder && formToMove) {

correctPlaceholder.insertAdjacentElement('afterend', formToMove);

}

}

hasBeenFixed = true;

if (observer) {

observer.disconnect();

}

}

};

const observer = new MutationObserver((mutationsList) => {

for (const mutation of mutationsList) {

if (mutation.type === 'childList' && mutation.addedNodes.length > 0) {

fixFormLocations();

}

}

});

observer.observe(firstContainer, { childList: true, subtree: true });

setTimeout(fixFormLocations, 2000);

})();

</script>
```
