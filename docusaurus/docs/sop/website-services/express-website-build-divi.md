## Developer: Individual Plugin Settings - The specific settings for Divi Booster, Flying Pages, Flying Scripts, OMGF, and One Click Accessibility have already been configured on the Website Express: AI Generated Website template. The only settings you’ll need to adjust are listed below: {#developer-individual-plugin-settings-the-specific-settings-for-divi-booster-flyi}

- **Flying Scripts** \- If you are using a Divi Child Theme and find issues, please do the following to troubleshoot:
       1. Turn off all modules except the ones you are actively using
       2. Take out the keywords and add them in one by one to find out what is causing the blocking

- **Preload link** \- this is used to insert the largest graphic element's html code to pre-load on the homepage.
       1. Make sure the latest Divi child theme is installed from [here](https://drive.google.com/drive/folders/1NKsv1bQwbfL9xPKEbrP0AdG2G2YGkfuL). _The latest version has the php code used to insert the preload link on the home page only, above the post content._**Also, delete the WPCode plugin if it's already been installed.**
       2. Navigate to Appearance > Theme File Editor > **functions.php**
          1. Edit the following code for an image (changing the href to a relative link):
             1. <link rel="preload" href=" /wp-content/uploads/2025/02/flower.webp" as="image" type="image/webp" />
          2. Edit the following code for a video (changing the href to a relative link):
             1. <link rel="preload" href="/wp-content/uploads/2024/12/Placeholder-Video-1.mp4" as="video" type="video/mp4" />
          3. If the hero section background on the Home page is an image, remove the video code. If the hero section background on the Home page is a video, remove the image code.  
**Example of preload image code used**  
![](/img/sop-website-services/express-website-build-divi/14.png)

- **Yoast SEO** \- Running through the First-Time Configuration should ensure that the settings are correct. However, you will need to check the following in Settings as well:
       1. Site Features - ensure that the XML Sitemaps feature is enabled.
       2. Site Basics - ensure the website and alternate website name are set to the business name, and add the tagline/slogan as well as the logo uploaded as the Site image.
       3. Site Representation - ensure the Organization name is set to the business name, and the logo is added as the Organization logo.
       4. Advanced > Breadcrumbs - add the business name as the Anchor text for the Homepage.

- **Gravity Forms** \- _Only to be used if we are not using the Vendasta CRM Form_
       1. Settings
          1. Ensure you add the password, which should be available in JumpCloud.
          2. If the form will be taking funds, ensure the denomination has been changed to the correct currency.
       2. [reCAPTCHA](https://www.google.com/recaptcha/admin/site/596855112) \- 
          1. Create a new reCAPTCHA account (password in JumpCloud) and add in the temp domain, the marketing services.dev domain and the branded domain. Make sure the account is set to V2.
             1. The reCAPTCHA account should be created via the website analytics account - get the UN/PW from JumpCloud
          2. Add the Site Key and the Secret Key to the Gravity Forms settings.
          3. As a note, V2 is usually our standard but if necessary (due to multiple forms on the same page), we can go one of two routes -
             1. reCAPTCHA Add-On (via Gravity Forms Add-Ons) - allows the V3 version which means that only one form on the page needs to have the reCAPTCHA module in the form.
             2. hCAPTCHA Add-On (via standard plugins) - this is the last defense for a site whose forms get spammed constantly as it is very hard for both bot and normal users to get past.
       3. Form Settings - these will be discussed later during the form creation section of the build.

## Developer: Theme Customizer Settings - Ensure that you change your settings here before starting the build. This will help maintain consistency across the website for font, colours, buttons, etc. {#developer-theme-customizer-settings-ensure-that-you-change-your-settings-here-be}

- **General Settings**
       1. **Site Identity**
          1. Ensure the site title is correct
          2. Add the tagline if not completed during the WSP launch
          3. Add the Favicon to the site title here
             1. It should be a png with a transparent background as well as square, no larger than 512x512. As a note, it should be a multiple of 48x48 according to Google Search Console Best Practices.
       2. **Layout Settings** \- most of these should be default settings except for the Primary and Secondary Colors, which should be the primary colour for the site and one of our set Color Palette options.
       3. **Typography**
          1. **Header Font Size** \- If you want the header to be all caps, italicized or anything like that, set that option here. That way it is consistent across the site.
          2. **Header Font & Body Font** \- we typically use Google Fonts or Adobe Fonts.
             1. If you are using Google Fonts, you should be able to find the fonts in the lists.
             2. If you are using Adobe Fonts, you will need to add a plugin such as Typekit Fonts for WordPress - found [here](https://wordpress.org/plugins/typekit-fonts-for-wordpress/). Or this plugin, Custom Adobe Fonts (Typekit) - found [here](https://wordpress.org/plugins/custom-typekit-fonts/).

[Here ](https://www.elegantthemes.com/blog/tips-tricks/how-to-add-typekit-fonts-to-wordpress)are the Divi directions for installing a Google Typekit.

      - If we are using a different font (that we MUST have a license for and is provided by the SMB/Client), we can use the Custom Fonts plugin - found [here](https://wordpress.org/plugins/custom-fonts/).
        1. **Body Link Color** \- it is best to use the primary site color, typically the same as the Theme Accent Color.
        2. **Text Color** \- either use the default or whatever is required for the general aesthetics of the site.
    1. **Buttons**
       1. Ensure that you make all your changes to the buttons in the Customizer so that they are consistent across the site.
          1. If you change a setting such as the border radius (for a round logo that you would like mimic) in the Buttons Style, then make sure you duplicate that radius in the Buttons Hover Style.
    2. **Header & Navigation**
       1. **Primary Menu Bar** \- 
          1. Try to ensure you use a logo with a transparent background (a png) or ensure that the background matches the edge of the logo to make it blend in.
          2. Try to use colours from the primary colours for the navigation links and hover links.
          3. Keep in mind that the dropdown menu colours will also need to change for the background if a dropdown is used. Try to keep it consistent.
       2. **Secondary Menu Bar**
          1. Try to use the Primary Colours again for the background.
       3. **Fixed Navigation** \- this will be the same as the Primary Menu Bar more often than not.
       4. **Header Elements** \- remember to add the correct phone number and email address for the site. And only if the phone number is serving an international customer base do you add the +Country Code.  

## Developer: Generate Website with AI {#developer-generate-website-with-ai}

- **Navigate to Divi > Dashboard on the back-end of the website**
    2. Click on the purple “**Generate A New Site** ” button  
![](/img/sop-website-services/express-website-build-divi/16.png)
    3. Click on the blue “**Generate My Website** ” button under Generate Your Site With AI  
![](/img/sop-website-services/express-website-build-divi/17.png)
    4. Fill out the prompts with the following information
       1. **Site Name:** The site name, according to the fulfillment form
       2. **Site Slogan:** If a site slogan is specified in the fulfillment form add it here, or use the site name if none was provided
       3. **Tell Divi AI About Your Website:** Make the necessary changes to this prompt template  
  
Generate a comprehensive, [# of pages]-page website for a [Type of business or purpose, e.g., "boutique coffee shop," "freelance graphic designer," "non-profit organization"]. The site should be fast, responsive, and easy to navigate.

**Business & Project Information**

[Provide a detailed text description of their business and services. Include their mission, values, what makes them unique. Briefly describe their ideal customer or target visitor, and state your primary call-to-action (e.g., "Book a Consultation," "Shop Now," "Get a Free Quote").]

**Business Contact Info:**

Phone Number: [e.g., (555) 123-4567]

Email Address: [e.g., [info@yourbusiness.com](mailto:info@yourbusiness.com)]

Hours of Operation: [e.g., Mon - Fri: 8am - 5pm]

Address: [e.g., 111 Fake Street]

**Website Structure & Content Directives**

Generate a complete, [# of pages]-page website with the following standard pages:

[Page name 1]

[Page name 2]

[Page name 3]

[Page name 4]

[Page name 5]

Use the information provided above to create unique, high-quality, and concise content for each page section. The AI should apply standard business best practices for content hierarchy, readability, and user flow. All headings must be used in a sequentially-descending order (H1, H2, H3, etc.) on each page.

**Footer Directives**

Include the provided phone number, email address, hours of operation, and address in the website footer.

**Design & Performance Directives**

Image Optimization: Crucially, ensure all images are compressed and lazy-loaded to maximize page speed. Use modern formats like WebP where possible.

Responsive Design: The entire site must be fully responsive and look excellent on all devices, especially mobile phones.

Lighthouse Performance: The website must be built for maximum speed and performance on both desktop and mobile, with a primary goal of a high score on Google's Lighthouse audit. This requires:

Crucially, ensuring a high color contrast ratio between all background and foreground elements for accessibility and readability.

Aggressive image optimization, including compression, proper sizing, and lazy loading.

**To run multiple CRM forms on the same page use below script!!**

```html
<script>

(function() {

const placeholders = document.querySelectorAll('script[id="__custom_form_widget"]');

if (placeholders.length <= 1) {

return;

}

const firstContainer = placeholders[0].parentElement;

if (!firstContainer) {

return;

}

let hasBeenFixed = false;

const fixFormLocations = () => {

if (hasBeenFixed) return;

const generatedForms = document.querySelectorAll('form.form-widget');

if (placeholders.length === generatedForms.length) {

for (let i = 1; i < placeholders.length; i++) {

const correctPlaceholder = placeholders[i];

const formToMove = generatedForms[i];

if (correctPlaceholder && formToMove) {

correctPlaceholder.insertAdjacentElement('afterend', formToMove);

}

}

hasBeenFixed = true;

if (observer) {

observer.disconnect();

}

}

};

const observer = new MutationObserver((mutationsList) => {

for (const mutation of mutationsList) {

if (mutation.type === 'childList' && mutation.addedNodes.length > 0) {

fixFormLocations();

}

}

});

observer.observe(firstContainer, { childList: true, subtree: true });

setTimeout(fixFormLocations, 2000);

})();

</script>
```
