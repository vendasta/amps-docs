VS Internal Bot: Knowledge Base & Feedback Loop

1. Overview

This document describes the end-to-end process for:

  1. Knowledge sourcing and delivery — Exporting content from Guru cards, publishing SOPs and knowledge in the AMPS Docs repository, connecting that knowledge to the Chat Receptionist – VS Internal Bot, and having the bot answer user queries in Google Chat.
  2. Feedback and correction loop — Allowing users to flag incorrect or contradictory bot responses by reacting with a ticket emoji (🎫). That reaction creates a row in a Google Sheet, which triggers creation of a Jira task. Tasks are assigned to knowledge owners to update the source content, keeping the bot's knowledge accurate over time.

Goals:

  • Single, up-to-date source of truth for SOPs and Guru-style knowledge (AMPS Docs).
  • Internal users get answers in the VS Internal Google Chat space from the VS Internal Bot.
  • Incorrect or conflicting answers are captured, turned into Jira tasks, and fixed at the source so the bot stays reliable.

2. Architecture Overview

Two flows work together: knowledge delivery (source → bot → users) and feedback loop (users flag errors → sheet → Jira → fix at source).

3. Part 1: Knowledge Base Setup and Bot Answers

3.1 From Guru Cards to AMPS Docs

  • Content was exported from Guru cards and used to create or update SOPs and knowledge articles in the AMPS Docs repository.
  • The repository is built and published as a documentation site.

The published knowledge base (AMPS Docs) is the single source of truth for the procedures and knowledge that the bot uses.

3.2 Connecting Knowledge to the Bot

  • The Chat Receptionist – VS Internal Bot is configured in Vendasta's Business App (AI Assistants).
  • Knowledge links pointing to the AMPS Docs site (and relevant sections) were added to this bot so it can use the published SOPs and Guru-derived content to answer questions.

Open the Chat Receptionist – VS Internal Bot edit page (Vendasta sign-in required) to manage knowledge links.

3.3 Where Users Get Answers

  • Users ask questions about SOPs and Guru card knowledge in the VS Internal Google Chat space.
  • The VS Internal Bot responds in that space using the knowledge linked from AMPS Docs.

Summary of Part 1:

StepWhat happens
1Guru card content is exported and used to create/update SOPs in the AMPS Docs repo.
2AMPS Docs is published at the published knowledge base.
3The Chat Receptionist – VS Internal Bot is given knowledge links to this published site via the bot edit page.
4Users ask questions in the VS Internal Google Chat space; the bot answers using that knowledge.

4. Part 2: Feedback Loop (Flag Wrong Answers → Sheet → Jira → Fix Knowledge)

This is the correction and maintenance process so the bot's answers stay accurate.

4.1 User flags a wrong or contradicting answer

  • When a user sees a bot response that is incorrect or contradicts known procedures/SOPs, they react to that chat message with the 🎫 (ticket) emoji.

4.2 New row on the Google Sheet

  • That reaction is processed so that a new row is created on the feedback/requests Google Sheet.
  • The row captures the context needed to investigate and fix the answer (e.g. query, bot response, thread/message reference).

4.3 New task on the Jira board

  • When a new row is added (or when the sheet is updated), automation runs (e.g. Apps Script + Jira webhook) and creates a new task on the Chat-Jira Test (CJT) Jira board.
  • The task represents "fix the knowledge that led to this wrong answer."

4.4 Assignment and update of knowledge

  • Tasks are assigned to the existing knowledge owners on the board (e.g. by area: SOPs, Guru content, product).
  • Owners update the knowledge source (AMPS Docs and/or Guru, as applicable) so that:
    • The corrected content is published in AMPS Docs.
    • The bot (which uses links to AMPS Docs) then responds correctly to similar questions in the future.

Summary of Part 2:

StepWhat happens
1User reacts with 🎫 on the bot's incorrect/contradicting message in the VS Internal Google Chat space.

3Automation creates a new task on the CJT Jira board.
4Task is assigned to the relevant knowledge owner(s).
5Owner updates the knowledge source (AMPS Docs / Guru); bot answers improve.

6. Summary

  • Knowledge flow: Guru → AMPS Docs (repo + published site) → knowledge links in Chat Receptionist – VS Internal Bot → answers in the VS Internal Google Chat space.
  • Feedback flow: User reacts 🎫 on wrong answer → row in feedback/requests Sheet → task on Jira (CJT) → owners update knowledge → bot responds correctly.

This keeps the VS Internal Bot aligned with current SOPs and Guru knowledge and ensures corrections are tracked and applied at the source.
